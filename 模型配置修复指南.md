# 模型配置修复指南

## 问题诊断

批改报告显示"批改过程中出现错误，已给予基础分10分"，原因是：

1. **本地模型调用失败**：系统默认使用本地Ollama模型，但Ollama服务未启动或模型未下载
2. **云端模型未配置**：智谱AI API key未配置，无法使用云端模型作为fallback

## 解决方案

### 方案一：使用智谱AI云端模型（推荐）

1. **获取智谱AI API Key**
   - 访问：https://open.bigmodel.cn/
   - 注册账号并获取API Key

2. **创建配置文件**
   - 在项目根目录创建 `.env` 文件
   - 复制 `.env.example` 的内容
   - 修改以下配置：
     ```
     USE_LOCAL_MODEL=False
     ZHIPU_API_KEY=你的API_Key
     ```

3. **重启后端服务**
   ```bash
   cd backend
   python -m uvicorn app.main:app --reload
   ```

### 方案二：使用本地Ollama模型

1. **安装Ollama**
   - 访问：https://ollama.ai/
   - 下载并安装Ollama

2. **下载模型**
   ```bash
   ollama pull qwen2.5:7b
   ```

3. **启动Ollama服务**
   ```bash
   ollama serve
   ```

4. **验证服务**
   ```bash
   curl http://localhost:11434/api/generate -d '{"model":"qwen2.5:7b","prompt":"hello","stream":false}'
   ```

5. **创建配置文件**
   - 在项目根目录创建 `.env` 文件
   - 复制 `.env.example` 的内容
   - 修改以下配置：
     ```
     USE_LOCAL_MODEL=True
     LOCAL_MODEL_URL=http://localhost:11434/api/generate
     LOCAL_MODEL_NAME=qwen2.5:7b
     ```

6. **重启后端服务**
   ```bash
   cd backend
   python -m uvicorn app.main:app --reload
   ```

## 验证修复

1. **检查模型状态**
   - 访问：http://localhost:8000/api/model/status
   - 确认显示正确的模型配置

2. **测试批改功能**
   - 登录学生端
   - 选择题目并提交代码
   - 查看批改报告是否完整显示

3. **检查日志**
   - 查看 `backend/logs/grade.log`
   - 确认没有错误信息
   - 确认模型返回内容不为空

## 常见问题

### Q1: 智谱AI API Key无效
- 检查API Key是否正确复制
- 确认API Key是否有效（未过期）
- 确认账户余额是否充足

### Q2: Ollama服务无法启动
- 检查Ollama是否正确安装
- 确认端口11434未被占用
- 查看Ollama日志排查问题

### Q3: 批改报告仍然显示错误
- 检查 `.env` 文件是否在正确位置（项目根目录）
- 确认配置文件格式正确（无多余空格或特殊字符）
- 重启后端服务使配置生效

## 技术细节

### Fallback机制

系统已添加智能fallback机制：
1. 首先尝试使用本地模型
2. 如果本地模型失败或返回空内容，自动切换到云端模型
3. 详细记录每一步的执行情况

### 日志记录

系统会记录以下关键信息：
- 使用的模型类型
- 模型返回内容的长度
- 清理后的内容长度
- JSON解析结果
- 错误详情（如果发生）

### 错误处理

当模型返回空内容时：
- 不再显示通用的"批改过程中出现错误"
- 提示"模型返回内容为空，请检查模型配置或网络连接"
- 建议用户检查模型配置
